== Zweck

SubscriptionGuard soll es ermöglichen Abonnements und laufenden Verträge einfach zu überblicken und zu verstehen, um finanzielle Transparenz und Kontrolle im Alltag zu gewinnen. Ziel ist die Entwicklung einer benutzerfreundlichen Anwendung, die wiederkehrende Kosten sichtbar macht und rechtzeitig an Fristen erinnert um unnötige Ausgaben zu vermeiden und finanzielle Entscheidungen zu unterstützen.

=== Definition of Ready (DoR)
_Eine User Story ist bereit für die Entwicklung, wenn:_

// Hier können die Kriterien individuell angepasst werden, z. B.:
* [ ] Die Akzeptanzkriterien sind klar und vollständig beschrieben

=== Definition of Done (DoD)
_Eine User Story gilt als fertig, wenn:_

// Hier können die Kriterien individuell angepasst werden, z. B.:
* [ ] Der Code wurde implementiert und in den Hauptbranch gemergt
* [ ] Unit- und Integrationstests wurden geschrieben und sind grün
* [ ] Der Code wurde von mindestens einer weiteren Person reviewt
* [ ] Die Funktionalität wurde im Staging-System getestet
* [ ] Dokumentation wurde aktualisiert (falls erforderlich)
* [ ] Der Product Owner hat die Story abgenommen
